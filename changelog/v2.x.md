## [v2.2.0] — 2026/03/15

### What changed since v2.1.0

**AmxString**
- Implements `Deref<Target = str>`: every `&str` method is available directly — `name.starts_with("x")`, `name.contains("y")`, `format!("{name}")`, etc.
- Lazy decoding through `OnceCell<String>`: the `String` is only allocated on the first `Deref` access, not at construction. Natives that never use the string as `&str` pay no cost.
- `PartialEq<str>`, `PartialEq<&str>`, `PartialEq<String>`: direct comparison against literals — `if name == "Admin"` with no conversion.
- `as_str() -> &str`: explicit alternative to `&*name` / `&name`, self-documenting for readers unfamiliar with `Deref`.

**Typed buffers**
- `CellConvert` trait: `i32 ↔ T` conversion without an AMX context — implemented for every primitive (`i8`, `u8`, `i16`, `u16`, `i32`, `u32`, `usize`, `isize`, `f32`, `bool`).
- `Buffer::get_as::<T>(index) -> Option<T>` — reads a cell converted into `T`; `None` when out of bounds.
- `Buffer::set_as::<T>(index, value) -> bool` — converts and writes; `false` when out of bounds.
- `Buffer::iter_as::<T>()` — typed iterator over every cell; composes with `sum`, `any`, `filter_map`, etc.
- No abstraction cost: Pawn `Float:` / `bool:` arrays accessible without manual bit manipulation.

**Output strings**
- `Buffer::write_str(&mut self, s: &str) -> AmxResult<()>` — direct replacement for `put_in_buffer`.
- `UnsizedBuffer::write_str(self, max_len, s) -> AmxResult<()>` — combines `into_sized_buffer` + write in one step, propagating errors via `?`.

**Plugin ergonomics**
- `#[derive(SampPlugin)]` emits `impl SampPlugin for T {}` for structs without overrides.
- `initialize_plugin!(type: T, natives: [...])` uses `Default::default()` as the constructor — removes the boilerplate block when there is no initialization logic.

**Examples**
- `example-hello` — minimal plugin: demonstrates `#[derive(SampPlugin)]`, `initialize_plugin!(type:)`, `AmxString` through `Deref`, and `UnsizedBuffer::write_str`.
- `example-counter` — stateful plugin: demonstrates a hand-written `impl SampPlugin` with `on_load` / `process_tick`, multiple natives, `Ref<i32>` for output by reference, and a full constructor block.
- `plugin-example` kept as the advanced reference (memcache, encoding, complete logging setup).

**Documentation**
- Migration guide v2.1.0 → v2.2.0 with before/after diffs for every API change.
- `docs/src/natives.md`: typed-array section clarifies that `get_as` / `set_as` do not require importing `CellConvert`; admonitions for lazy `AmxString`, `write_str`, and the `AmxCell` vs. `CellConvert` split.
- `docs/src/anatomia-plugin.md`: documents the two forms of `initialize_plugin!` with a table of when to use each.
- `docs/src/primeiro-plugin.md`: examples updated to the new API.

**CI/CD**
- `bench` job with a persistent historical baseline: every run is stored through `actions/cache`, compared with `critcmp` (5% threshold), and regressions are reported in the log.

**Tests and benchmarks**
- 59 unit tests + 24 doctests (previously 32 + 5).
- Criterion benchmarks for unpacked, packed, `write_str`, lazy vs. cached `Deref`, and an AMX-less baseline.

## [v2.1.0] — 2026/02/23

### Security

- **[CRITICAL]** Fixed an off-by-one in `Args::get()` that allowed out-of-bounds reads (CWE-125).
- **[CRITICAL]** Added a null-pointer check in `Amx::get_ref()` after the FFI call (CWE-416).
- **[HIGH]** Guarded `Amx::allot()` against integer overflow by validating `cells > i32::MAX` (CWE-190).
- **[HIGH]** Added bounds checking in packed-string parsing to prevent reads past the buffer (CWE-125).
- **[HIGH]** Fixed `AtomicPtr` ordering from `Relaxed` to `Release` / `Acquire` in `encoding.rs` (CWE-362).
- **[MEDIUM]** Added `debug_assert!` for null / misaligned pointers in `Ref::new()` (CWE-843).
- **[MEDIUM]** Added address validation in `Amx::release()` before mutating the heap (CWE-763).
- **[MEDIUM]** Added function-pointer validation in `from_table()` of the exports module (CWE-476).
- **[MEDIUM]** Eliminated undefined behavior in `Runtime` via `UnsafeCell<RuntimeInner>` (CWE-362).
- **[MEDIUM]** Capped `UnsizedBuffer::into_sized_buffer()` at 1 MiB (CWE-120).
- **[LOW]** Guarded `Args::count()` against negative values (CWE-190).
- **[LOW]** Fixed `AmxString::new()` to use `bytes.len()` instead of `buffer.len()` (CWE-170).
- **[LOW]** Capped string allocations at `MAX_STRING_LEN = 1 MiB` against OOM (CWE-20).
- **[LOW]** Made the native-name memory leak explicit through `Box::leak()` (CWE-401).

### Tests

- Added `Args` tests: `count_with_zero`, `count_with_negative`, `count_with_valid_args`, `get_out_of_bounds`, `reset`.
- Added `encoding` tests: `default_encoding_is_windows_1252`, `set_and_get_encoding`.
- Added `AmxError` tests: `memory_access_display`, `memory_error_display`, `amx_result_ok`, `amx_result_err`.
- Total: 32 unit tests + 27 doctests.

### CI/CD

- Added `cargo audit` to the workflow to catch CVEs in dependencies.

### Internal

- `Runtime` refactored to `UnsafeCell<RuntimeInner>` for safe interior mutability.
- `Runtime::get()` now returns `&'static Runtime` (immutable) instead of `&'static mut`.
- Mutating `Runtime` methods now take `&self` and use interior mutability.
- `CString::into_raw()` in natives replaced by `Box::leak(CString.into_boxed_c_str())`.

### Versions

- `samp-sdk`: 2.0.0 → 2.1.0
- `samp`: 2.0.0 → 2.1.0

## [v2.0.0] — no published tag (folded into v2.1.0)

### CI/CD

- Added cross-platform i686 builds in GitHub Actions (Linux + Windows).
- Configured `Swatinem/rust-cache` for dependency caching.
- Added `cargo clippy -D warnings` to the workflow.
- Artifact uploads (`.so` / `.dll`) on pushes to `master`.

### Error handling

- Pointer validation in the SDK's `unsafe` blocks.
- Procedural macros emit error messages in Portuguese (legacy — switched to English in later releases).
- `Result` adopted across more of the public API.

### Tests

- Unit tests for `AmxPrimitive`, `AmxError`, `consts`, `Exports`.
- Doctest fixes (`compile_fail` → `no_run` where appropriate).
- 21 unit tests + 27 doctests.

### Documentation

- Full mdBook documentation (13 chapters in Portuguese).
- Guides: introduction, first plugin, anatomy, natives, AMX types, cells / memory, encoding, errors, logging, advanced examples, migration, API reference.
